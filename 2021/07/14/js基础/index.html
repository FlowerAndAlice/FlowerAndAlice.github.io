变量声明

变量名可以是字母、下划线、$ 和 数字，但不能以数字开头。 (驼峰)

  • 保留字：Abstract、Byte、Char、Debugger、Export、Static…

JavaScript是单线程语言，所以执行肯定是按顺序执行。但是并不是逐行的分析和执行，而是一段一段地分析执行，会先进行编译阶段然后执行。在编译阶段，代码真正执行前的几毫秒，会检测到所有的变量和函数声明，所有这些函数和变量声明都被添加到名为Lexical Environment的JavaScript数据结构内的内存中。所以这些变量和函数能在它们真正被声明之前使用。

  • var ——ES5 变量声明方式
  1. 在变量未赋值时，变量undefined（包括变量提升）
  2. 作用域——var的作用域为方法作用域；只要在方法内定义了，整个方法内的定义变量后的代码都可以使用
  3. 在块内声明函数或变量,只要没加var,都算作全局函数/变量 ( 且严格模式会报错 ) ( 函数形参除外 ) (又有传入引用类型时拿到的是全局中的对象的引用)
  • let——ES6变量声明方式

  1. 在变量为声明前直接使用会报错（无变量提升）

  2. let为块作用域—— ( 大括号中使用let/const才是块级,var全局 )

    let在代码块中有自己的作用域，( 只有该块级作用域中有i,且该i在循环内 )

    所以for循环中的let声明的每一个值都会单独存在于一个作用域中,不会被覆盖掉。

  3. let禁止重复声明变量，否则会报错；var可以重复声明

  • const——ES6变量声明方式

  1. const为常量声明方式；声明变量时必须初始化，在后面出现的代码中不能再修改该常量的值( 一般在声明函数表达式和对象时使用const )

  2. const实际上保证的，并不是变量的值不得改动，而是变量指向的那个内存地址不得改动,所以想要修改const定义的变量时,可将其定义为对象或数组[ ] , 通过arr[0]调用/修改。 (对象同理)

this 与 arguments与严格模式

解释器在调用函数每次都会向函数内部传进俩个隐含的参数。

  • 一个隐含的参数就是this，是一个引用, 指向内存中的一个对象

    • 这个对象称之为函数执行的上下文对象，根据函数的调用方式不同，this指向不同的对象

  • 一个隐含的参数就是封装实参的对象arguments

    • arguments是一个类数组对象，他可以通过索引来操作数据，也可以获取长度
    • 在调用函数时，我们所传递的实参都会封装到arguments保存

  • this指向

    • 以函数的形式调用( 包括立即执行函数、匿名函数 )，this永远都是window ( window.fun ,函数都是window的对象 )
    • 以**方法的形式( 对象的属性 )**调用时，this就是调用方法的那个对象
    • 以构建函数的形式调用时，this就是新创建的那个( 实例 )对象
    • 使用call和apply调用时，this指向传入的第一个参数
    • 箭头函数中的this , 指向函数定义时的上下文的this( 箭头函数与普通函数的区别 ),且call()等无法改变this指向,只能传参
    • 可以在上下文中把this传给变量,在函数中使用该变量,也可改变this指向
    • 函数定义时若处于严格模式下,this会指向undefined,箭头函数不受影响
      • “use strict” : 在正常模式中，如果一个变量没有声明就赋值，默认是全局变量。严格模式必须显式声明全局变量,不能使用未声明的变量。
        • 将”use strict”放在脚本文件的第一行，则整个脚本都将以”严格模式”运行。
        • 将”use strict”放在函数体的第一行，则整个函数以”严格模式”运行。
      • 禁止this关键字指向全局对象

  • call, bind,apply

    call是一个方法,是函数的方法

  1. call、apply可以调用函数

  2. call的第一个参数可以改变函数中this指向,后面的**(多个)参数**是传给函数的参数

  3. apply功能一样,但必须传两个参数,新this对象和一个参数数组argArray

  4. bind不能调用函数，会返回一个函数，执行后得到和call相同的结果,传递值和数组或不传值都可以

  5. call应用: 子类继承父类的方法 (多重)

同步异步

  • 单线程 : Js是一门单线程语言,同一时间只能做一件事,如果有多个任务的话,这些任务就需要进行排队,前一个任务执行完,才会执行下一个任务
  • 为什么是单线程 : JavaScript作为浏览器的脚本语言，主要用来实现与用户的交互，利用JavaScript，我们可以实现对DOM的各种各样的操作，如果JavaScript是多线程的话，一个线程在一个DOM节点中增加内容，另一个线程要删除这个DOM节点，那么这个DOM节点究竟是要增加内容还是删除呢？这会带来很复杂的同步问题，因此，JavaScript是单线程的
  • 同步任务 : 同步任务是指在主线程上排队执行的任务，只有前一个任务执行完毕，才能继续执行下一个任务，当我们打开网站时，网站的渲染过程，比如元素的渲染，其实就是一个同步任务
  • 异步任务 : 异步任务是指不进入主线程，而进入任务队列的任务，只有任务队列通知主线程，某个异步任务可以执行了，该任务才会进入主线程，当我们打开网站时，像图片的加载，音乐的加载，其实就是异步任务( 先执行一部分,等拿到结果或者时间到了后 再执行后续代码 )

Event-Loop(浏览器)

  • event loop是一个执行模型，在不同的地方有不同的实现。浏览器和NodeJS基于不同的技术实现了各自的Event Loop。

  • 进程：运行的程序就是一个进程，比如你正在运行的浏览器，它会有一个进程；

  • 线程：程序中独立运行的代码段。一个进程 由单个或多个 线程 组成，线程是负责执行代码的。

  • 单线程：从头执行到尾，一行行执行，如果其中一行代码报错，那么剩下的代码将不再执行。同时容易代码阻塞。

  • 多线程：代码运行的环境不同，各线程独立，互不影响，避免阻塞。

为了保证 JavaScript 的主要运行环境（浏览器），JavaScript 被设计成了单线程语言。

为了充分利用多核处理器的计算能力，HTML5 提出了 Web Worker 标准，允许 JavaScript 脚本创建多个子线程，但是子线程完全受控于主线程，且不得操作 DOM。因此，新标准并没有改变 JavaScript 单线程的本质。

  • call stack( 调用栈 ) : Event Loop 会一直检查 Call Stack 中是否有函数需要执行，如果有，就从栈顶依次执行。同时，如果执行的过程中发现其他函数，继续入栈然后执行。

    栈空
    现在执行到一个函数A，函数A入栈
    函数A又调用了函数B，函数B入栈
    函数B执行完后出栈 ( 后入先出 )
    然后继续执行函数A，执行完后A也出栈
    栈空

  • Event Table 可以理解成一张 事件->回调函数 对应表

    它就是用来存储 JavaScript 中的异步事件 (request, setTimeout, IO等) 及其对应的回调函数的列表

  • Event Queue 简单理解就是 回调函数 队列，所以它也叫 Callback Queue

    当 Event Table 中的事件被触发，事件对应的 回调函数 就会被 push 进这个 Event Queue，然后等待被执行

  • Event Loop

  1. 开始，任务先进入 Call Stack

  2. 同步任务直接在栈中等待被执行，异步任务从 Call Stack 移入到 Event Table 注册

  3. 当对应事件触发（或到指定时间），Event Table 会将事件回调函数移入 Event Queue 等待

  4. 当 Call Stack 中没有任务，就从 Event Queue 中拿出一个任务放入 Call Stack

  • Event Loop 指的就是这一整个圈圈：

它不停检查 Call Stack 中是否有任务（也叫栈帧）需要执行，如果没有，就检查 Event Queue，从中弹出一个任务，放入 Call Stack 中，如此往复循环。

Event Queue在 HTML 规范中被称为 Task Queue，但是为了区分，一般都叫作 Macrotask Queue
Job Queue 是在 ECMAScript 规范中谈及处理 Promise 回调时提到的，但是由于和 V8 中的实现比较相似，所以一般都称为 Microtask Queue

  • Macrotasks ( 宏任务)

    Macrotasks 包含了解析 HTML、生成 DOM、执行主线程 JS 代码和其他事件如 页面加载、输入、网络事件、定时器事件等。从浏览器的角度，Macrotask 代表的是一些离散的独立的工作。

    常见应用
    ①整体代码script ②setTimeout ③setInterval ④ajax ⑤读取文件 UI rednding

    • (setTimeout这个函数，是经过指定时间后，把要执行的任务加入到Event Queue中)

    • (setTimeout(fn,0)的含义是，指定某个任务在主线程最早可得的空闲时间执行，等待最低的4毫秒后，只要主线程执行栈内的同步任务全部执行完成，栈为空就马上执行 )

    • setInterval(fn,ms) : 每过ms秒,将注册的函数置入Event Queue，如果前面的任务耗时太久，那么同样需要等待。

  • Microtasks ( 微任务 )

    Microtasks 则是为了完成一些更新应用程序状态的较小的任务，如处理 Promise 的回调和 DOM 的修改，以便让这些任务在浏览器重新渲染之前执行。Microtask 应该以异步的方式尽快执行，所以它们的开销比 Macrotask 要小，并且可以使我们在 UI 重新渲染之前执行，避免了不必要的 UI 渲染。

    常见应用
    ①process.nextTick ②Promise.then

    • process.nextTick(callback)类似node.js版的”setTimeout”，在事件循环的下一次循环中( 整体同步代码执行完后 )调用 callback 回调函数。

  • 执行顺序

    Event Loop 的实现需要至少一个 Macrotask Queue 和至少一个 Microtask Queue。为了便于理解，我们都简化成一个。
    简单来说，Microtask Queue 具有更高的优先级，即执行一个 Macrotask 任务后，就会清空整个 Microtask Queue，此时如果有新的 Microtask 加入也会被执行。

    ①同步(宏任务) ②process.nextTick ③微任务 ④setImmediate ( 本次事件循环结束后执行 ) ⑤宏任务(下一轮)

数据类型

  1. js数据类型 ：

    基本数据类型 ：

    Undefined，Null，Number，String，Boolean,**symbol(ES6)、bigint(ES10)**。

    引用(复杂)数据类型：

    Object，function（proto Function.prototype）

    Object：普通对象，数组对象，正则对象，日期对象，Math数学函数对象。

  2. 基本类型值可以被替换，但不能被改变，赋值行为可以给基本类型一个新值，而不是改变它。调用字符串的方法，不会影响之前的字符串；调用数组的方法，部分会影响之前的数组。

  3. 堆和栈的区别

    （基本类型：String, Number, Boolean, Null，Undefined, Symbol, Bigint）

    原始(基本)数据类型直接存储在栈（stack）中简单数据段，占据空间小，大小固定，属于被频繁使用的数据，所以存储在栈中；

    （引用类型：object）

    引用数据类型直接存储在堆 (heap) 中，占据空间大，大小不固定，如果存储在栈中，将会影响程序运行的性能，引用数据类型在栈中存储了指针，该指针指向堆中该实体的起始地址，当解释器寻找引用值时，会首先检索其在栈中的地址，取得地址后，从堆中获得实体。

    两种数据类型的区别：

    1. 堆比栈空间大，栈比堆运行速度快。
    2. 堆内存是无序存储，可以根据引用直接获取。
    3. 基础数据类型比较稳定，而且相对来说占用的内存小。
    4. 引用数据类型大小是动态的，而且是无限的。

  4. 基本类型（实例）的 __proto__ 属性会指向对应包装对象的原型

    实例的constructor指向构造函数

    在包装对象的原型上有基本类型的方法

  5. Symbol 内部没有construtor构造器,不能用new关键字创建

原型链

  • ①__proto__和constructor属性是对象所独有的；

    ② prototype属性是函数所独有的，因为函数也是一种对象，所以函数也拥有__proto__和constructor属性。( 函数的proto都是Function.prototype )

  • ___proto_属性的作用就是当访问一个对象的属性时，如果该对象内部不存在该属性，那么就会去它的__proto__属性指向的那个对象（它的原型对象）里找，一直找，直到proto属性的终点null，再往上找就相当于在null上取值，会报错。通过proto属性将对象连接起来的这条链路即我们所谓的原型链。构造函数新建的实例内的proto指针（继承自构造函数的prototype），指向构造函数的原型对象。

    注意: Function.proto === Function.prototype

    ​ Object.proto === Function.prototype

  • Object.prototype 是一切对象的原型,也就意味着它不能有任何对象作为原型。

    综上，Object.prototype.__proto__ 只能为 null，意为这里不该有值。

    这一定义没有「用」，却对于 Javascript 的逻辑上完备而言是必须的。

  • prototype属性的作用就是让该函数所实例化的对象们都可以找到公用的属性和方法，即foo.__proto__ === Foo.prototype。

  • constructor属性的含义就是指向该( 原型 )对象的构造函数，Function这个对象比较特殊，它的构造函数就是它自己（因为Function可以看成是一个函数，也可以是一个对象），所有函数和对象最终都是由Function构造函数得来，所以constructor属性的终点就是Function这个函数。没有constructor属性的可以通过原型链向下查找到原型中的constructor属性并继承这个constructor属性( 指向与原型相同 )

  • 获取原型( 对象)的方法:

    1. 通过**对象的_proto_**获取
    2. 通过实例的构造函数的prototype属性获取 ( 构造函数是大写的那个 )

  • ES6 与 ES5 继承的区别

    1. 在 ES6 中类 B 继承了类 A 的属性，在 ES5 中，构造函数 B 没有继承构造函数 A 的属性，这是 ES6 与 ES5 继承的第一个区别。
    2. 在继承原生构造函数的情况下,通过 super()，在 ES6 中，新的实例具有原生构造函数实例的特性。在 ES5 中， 新实例无法获得原生构造函数原生构造函数实例的内部属性，通过 call(this) 也不行。

异步编程

return只能返回同步数据, 异步中获取数据的方法有3种

  • 回调函数(将函数作为参数传递给另一个函数,也就是将函数作为参数进行操作):

let target = “hello Yoimiya”;
function getData(fn) {
setTimeout(() => {
fn(target);
}, 500)
}
let result = getData((d) => {
console.log(d);
});
//维护困难

  • 用promise对象获取数据
    这是es6提供的一个内置对象,new Promise(); //专门解决异步问题

let target = “hello Yoimiya2”;
function getData() {
return new Promise((resolve) => {
setTimeout(() => {
resolve(target);
}, 666);
}); //then的函数参数的形参就是resolve传出来的值
}
let p = getData();
p.then((d) => {
console.log(d);
})

  • 用async函数

let target = “hello Yoimiya3”;
function getData() {
return new Promise((resolve) => {
setTimeout(() => {
resolve(target);
}, 666);
}); //then的函数参数的形参就是resolve传出来的值
}
async function pray() {
let data = await getData();
console.log(data);
}
pray();
//直接就获取了promise中的值,不用then,可以一个个获取
