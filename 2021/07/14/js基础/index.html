变量声明

变量名可以是字母、下划线、$ 和 数字，但不能以数字开头。 (驼峰)

  • 保留字：Abstract、Byte、Char、Debugger、Export、Static…

  • var ——ES5 变量声明方式

  1. 在变量未赋值时，变量undefined（包括变量提升）
  2. 作用域——var的作用域为方法作用域；只要在方法内定义了，整个方法内的定义变量后的代码都可以使用
  • let——ES6变量声明方式

  1. 在变量为声明前直接使用会报错（无变量提升）

  2. let为块作用域—— ( 大括号中使用let/const才是块级,var全局 )

    let在代码块中有自己的作用域，( 只有该块级作用域中有i,且该i在循环内 )

    所以for循环中的let声明的每一个值都会单独存在于一个作用域中,不会被覆盖掉。

  3. let禁止重复声明变量，否则会报错；var可以重复声明

  • const——ES6变量声明方式

  1. const为常量声明方式；声明变量时必须初始化，在后面出现的代码中不能再修改该常量的值

  2. const实际上保证的，并不是变量的值不得改动，而是变量指向的那个内存地址不得改动,所以想要修改const定义的变量时,可将其定义为数组[ ] , 通过arr[0]调用/修改。 (对象同理)

数据类型

  1. js数据类型 ：

    基本数据类型 ：

    Undefined，Null，Number，String，Boolean,**symbol(ES6)、bigint(ES10)**。

    引用(复杂)数据类型：

    Object，function（proto Function.prototype）

    Object：普通对象，数组对象，正则对象，日期对象，Math数学函数对象。

  2. 基本类型值可以被替换，但不能被改变，赋值行为可以给基本类型一个新值，而不是改变它。调用字符串的方法，不会影响之前的字符串；调用数组的方法，部分会影响之前的数组。

  3. 堆和栈的区别

    （基本类型：String, Number, Boolean, Null，Undefined, Symbol, Bigint）

    原始(基本)数据类型直接存储在栈（stack）中简单数据段，占据空间小，大小固定，属于被频繁使用的数据，所以存储在栈中；

    （引用类型：object）

    引用数据类型直接存储在堆 (heap) 中，占据空间大，大小不固定，如果存储在栈中，将会影响程序运行的性能，引用数据类型在栈中存储了指针，该指针指向堆中该实体的起始地址，当解释器寻找引用值时，会首先检索其在栈中的地址，取得地址后，从堆中获得实体。

    两种数据类型的区别：

    1. 堆比栈空间大，栈比堆运行速度快。
    2. 堆内存是无序存储，可以根据引用直接获取。
    3. 基础数据类型比较稳定，而且相对来说占用的内存小。
    4. 引用数据类型大小是动态的，而且是无限的。

  4. 基本类型（实例）的 __proto__ 属性会指向对应包装对象的原型

    实例的constructor指向构造函数

    在包装对象的原型上有基本类型的方法

  5. typeof 返回的是string，表示的是数据类型。

    • typeof Object：结果是”function”，因为Object是一个构造函数，可以通过new操作符来构造对象。
    • typeof null：结果是”object”，null是一个只有一个值的特殊类型。表示一个空对象引用。

  6. 转换为string：var a= a.toString(); 或者a=string(a);

  7. 将srting转为number(string有一些number没有的方法)：a=number(a)或者直接 +number(a);

  • map和forEach的区别 :

    相同点 :

    1. 都是循环遍历数组中的每一项

    2. 每次执行匿名函数都支持三个参数，参数分别为 item，index，arr

    3. 匿名函数中的this都是指向window

    4. 只能遍历数组

原型链

  • ①__proto__和constructor属性是对象所独有的；

    ② prototype属性是函数所独有的，因为函数也是一种对象，所以函数也拥有__proto__和constructor属性。( 函数的proto都是Function.prototype )

  • ___proto_属性的作用就是当访问一个对象的属性时，如果该对象内部不存在该属性，那么就会去它的__proto__属性指向的那个对象（它的原型对象）里找，一直找，直到proto属性的终点null，再往上找就相当于在null上取值，会报错。通过proto属性将对象连接起来的这条链路即我们所谓的原型链。构造函数新建的实例内的proto指针（继承自构造函数的prototype），指向构造函数的原型对象。

    注意: Function.proto === Function.prototype

    ​ Object.proto === Function.prototype

  • Object.prototype 是一切对象的原型,也就意味着它不能有任何对象作为原型。

    综上，Object.prototype.__proto__ 只能为 null，意为这里不该有值。

    这一定义没有「用」，却对于 Javascript 的逻辑上完备而言是必须的。

  • prototype属性的作用就是让该函数所实例化的对象们都可以找到公用的属性和方法，即foo.__proto__ === Foo.prototype。

  • constructor属性的含义就是指向该( 原型 )对象的构造函数，Function这个对象比较特殊，它的构造函数就是它自己（因为Function可以看成是一个函数，也可以是一个对象），所有函数和对象最终都是由Function构造函数得来，所以constructor属性的终点就是Function这个函数。没有constructor属性的可以通过原型链向下查找到原型中的constructor属性并继承这个constructor属性( 指向与原型相同 )

  • 获取原型( 对象)的方法:

    1. 通过**对象的_proto_**获取
    2. 通过实例的构造函数的prototype属性获取 ( 构造函数是大写的那个 )

  • ES5的继承 （ES6的在对象那节）

  • ES6 与 ES5 继承的区别

  1. 在 ES6 中类 B 继承了类 A 的属性，在 ES5 中，构造函数 B 没有继承构造函数 A 的属性，这是 ES6 与 ES5 继承的第一个区别。
  2. 在继承原生构造函数的情况下,通过 super()，在 ES6 中，新的实例具有原生构造函数实例的特性。在 ES5 中， 新实例无法获得原生构造函数原生构造函数实例的内部属性，通过 call(this) 也不行。

  • new Object()会创建一个继承自Object的对象；Object.create(Object.prototype)是凭空创建一个“新”对象并把新对象内部的proto关联到Object对象的prototype

    （ 不包含原型对象的属性及方法）

异步编程

return只能返回同步数据, 异步中获取数据的方法有3种

  • 回调函数(将函数作为参数传递给另一个函数,也就是将函数作为参数进行操作):

let target = “hello Yoimiya”;
function getData(fn) {
setTimeout(() => {
fn(target);
}, 500)
}
let result = getData((d) => {
console.log(d);
});
//维护困难

  • 用promise对象获取数据
    这是es6提供的一个内置对象,new Promise(); //专门解决异步问题

let target = “hello Yoimiya2”;
function getData() {
return new Promise((resolve) => {
setTimeout(() => {
resolve(target);
}, 666);
}); //then的函数参数的形参就是resolve传出来的值
}
let p = getData();
p.then((d) => {
console.log(d);
})

  • 用async函数

let target = “hello Yoimiya3”;
function getData() {
return new Promise((resolve) => {
setTimeout(() => {
resolve(target);
}, 666);
}); //then的函数参数的形参就是resolve传出来的值
}
async function pray() {
let data = await getData();
console.log(data);
}
pray();
//直接就获取了promise中的值,不用then,可以一个个获取
