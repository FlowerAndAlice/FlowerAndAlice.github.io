长度单位

  • 绝对长度单位

    ① px : 一般指CSS逻辑像素,它是相对于设备像素的单位，像素是相对于屏幕分辨率的，页面按照精确像素展示，不会因为其它元素的尺寸变化而变化，像素仍然是最典型的度量单位，例如：windows的用户所使用的分辨率96像素/英寸，而mac的用户所使用的分辨率一般是72像素/英寸，一般javascript语言里的单位就是使用的像素。

    可以设定固定的布局和元素的大小,但同时缺点为没有弹性

    ② in：英寸（inch）是一个物理度量文件，但是在CSS领域，英寸只不过被直接映射成像素罢了。（1in == 2.54cm == 96px）

    ③ em : 是相对于当前父元素的字体大小为标准

    ④ rem : 是相对于 html 元素的字体大小为标准

    ⑤ vw : 可视宽度单位（视窗宽度），1vw 等于可视区宽度的 1%

    ⑥ vh : 相对于可视化区域高度的1%

    所有的单位都会转为px,所以无论相对还是绝对,都是会变的

    在不同的分辨率下 ，像素点的大小是不同的，所以同一个网页，以 px 作长度单位时，在不同的分辨率下显示的大小是不同的，在低分辨率下，像素点较大，细节不够清晰，虽然显示的页面也大，但模糊不清。

    我们可以这样理解，将px看做绝对单位（显示上的绝对单位），其它的单位都是以 它为基础的，比如em，它就是相对当前文本字体的高度（假设当前文本字体的尺寸是12px，我们设置新的字体为1.5em，则新的字体尺寸转换为12 * 1.5 = 18px）

    如果 padding、 margin 设置了百分比，上，下，左，右 用的都是父元素宽度 的值为标准去计算。

CSS 选择器

  • 选择器权重

  1. 相同选择器: 后面的会覆盖前面的 ( 权重相同 )

  2. 不同选择器: ID(100) > CLASS(10) > element(1)

  3. 层级选择器: 按权重累加计算 (10+100>100+1)

  4. 最高权重: !important

  • 高级选择器
  1. 后代选择器 : 全部后辈 (子孙) 中的 body p{ background:pink; }
  2. 子代选择器 :一代,儿子辈的 body>p{ background：pink; } ( 空格是子代 , 逗号是一起 )

CSS定位(position)

  • static,默认值,通常是在覆盖absolute或者relative样式时使用。
  • fixed 相对浏览器的窗口进行定位。
  • relative 相对定位。(比如学生站队，教练喊小明。小明出队，向前走了一步。)
  • absolute 绝对定位
  1. 相对于最近的一个应用absolute或者relative的父元素，然后相对这个元素进行定位
  2. 它也是一种相对定位，不过相对的不再是自己，而是父元素。具体是哪个父元素，就要看父元素是否应用了position属性。
  3. 由此可以看出，如果父元素没有应用position样式，那么就会继续向上查找，直到html元素。如果都没有找到，absolute就会根据html进行定位；相反，如果找到某个父元素有absolute或者relative或者fixed，则会先对这个元素进行定位。
  • 通过非静态(非static)让元素非常独立可控,比如弹窗、下拉菜单、固定导航栏等场景

正常文档流

  • 默认的,按照书写顺序从上往下排,每个块级元素都会自动换行.如果有margin就会被隔开
    (p标签有默认margin)

脱离文档流

  1. 文档流: 将窗体自上而下分成一行一行，并在每行中按从左至右依次排放元素，称为文档流，也称为普通流。
    HTML中全部元素都是盒模型，盒模型占用一定的空间，依次排放在HTML中，形成了文档流。

  2. 元素脱离文档流之后，将不再在文档流中占据空间，而是处于浮动状态（可以理解为漂浮在文档流的上方）。
    脱离文档流的元素的定位基于正常的文档流，当一个元素脱离文档流后，依然在文档流中的其他元素将忽略该元素并填补其原先的空间

  • 方法:
  1. 使用float脱离文档流时，其他盒子会无视这个元素，但其他盒子内的文本认为这个元素依旧存在,会为这个元素让出位置
  2. 使用absolute脱离文档流后的元素，是相对于该元素的父元素进行定位的;
    并且这个父类元素的position必须是非static定位的
  3. fixed: 完全脱离文档流，相对于浏览器窗口进行定位。（相对于浏览器窗口就是相对于html）。

浮动布局

  • HTML元素分类:
  1. 块元素(可以设置宽度和高度,独立成行):       h1-6、p、div、ul、li
  2. 行内元素(内联元素): (不可以设置宽度和高度,不独立成行):   a、span
  3. 行内块元素(可以设置宽度和高度,独立成行)： img、imput、button
  4. 以上通过display属性实现： block、inline、inline-block，还有一个none实现隐藏

清除浮动

原因: 浮动溢出:容器的高度不能自动伸长以适应内容的高度，使内容溢出到容器外而影响布局的现象。

方法:

  1. 使用带clear属性的空元素 :
    在浮动元素后使用一个空元素如 <div class="clear"></div>
    并在CSS中赋予.clear{clear:both;}属性即可清理浮动。
    亦可使用<br class="clear" />或<hr class="clear" />来进行清理。
  2. 使用CSS的overflow属性:
    给浮动元素的容器(浮动的它的父级)添加overflow:hidden;或overflow:auto;可以清除浮动
    因为overflow.hidden会触发BFC。 BFC的意思是，我这个元素里面的子孙元素，不会影响外部元素的布局。
    故该容器里需全为浮动的元素
  3. 给浮动的元素的容器添加浮动:
    给浮动元素的容器也添加上浮动属性即可清除内部浮动，但是这样会使其整体浮动，影响布局，(不推荐使用)
  4. 使用邻接元素处理:
    给浮动元素后面的元素添加clear属性。
  5. 使用CSS的:after伪元素:
    在元素末尾添加一个内容为一个点并带有 clear: both 属性的元素

Flex(弹性)布局

  • 弹性盒子是一种用于按行或者按列布局元素的一维布局方法

  • 设为 Flex 布局以后，子元素的 float、clear和vertical-align属性将失效。要设置一个元素采用 flex 布局，只需设置器 display 属性取值为 flex 即可。

  • 采用 Flex 布局的元素，称为Flex容器。其子元素为flex item，本文称其为”项目”。容器默认存在两根轴：水平的主轴（main axis）和垂直的交叉轴（cross axis）。

    容器的属性

    • flex-direction : flex-direction属性决定主轴的方向（即项目的排列方向）。
      1. row（默认值）：主轴为水平方向，起点在左端。
      2. row-reverse：主轴为水平方向，起点在右端。
      3. column：主轴为垂直方向，起点在上沿。
      4. column-reverse：主轴为垂直方向，起点在下沿。
    • **flex-wrap **: 默认情况下，项目都排在一条线（又称”轴线”）上。flex-wrap属性定义，如果一条轴线排不下，如何换行。
      1. nowrap（默认）：不换行。
      2. wrap：换行，第一行在上方。
      3. wrap-reverse：换行，第一行在下方。
    • flex-flow : flex-flow属性是flex-direction属性和flex-wrap属性的简写形式，默认值为row nowrap。
    • justify-content : justify-content属性定义了项目在主轴上的对齐方式。
    • align-items : align-items属性定义项目在交叉轴上如何对齐。

动画

  • CSS动画 transfrom:(加对应动画)
    (过渡动画: dom加载完成或者class发生变化时触发)
  1. Translate(移动) : 该变换由二维向量构成,第一个参数是x轴移动距离,可用百分比和px
  2. scale(缩放) , (两参数： x、y),小于1缩小(大于1放大),
  3. rotate(旋转) 中心点默认在center,单位是deg
  4. skew(倾斜) : (与屏幕对比)(第一个参数水平方向,二是垂直)(单位deg)
  • keyframe实现动画
    (通过在动画序列中定义关键帧的样式来控制CSS动画序列中的中间步骤)
  1. from {} to {}
  2. 百分比
    0% {}
    50% {}
    100% {}
  3. 通过animation:(keyframes名称)可实现动画及设置属性

  • 优点: 简单高效,声明式,不依赖主线程,采用硬件加速(GPU),简单控制keyframe、animation播放暂停

  • 缺点： 不能动态修改或定义动画内容，不同的动画无法实现同步，多个动画彼此无法堆叠
    适用于简单的H5活动、宣传页
