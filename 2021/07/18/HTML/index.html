错题

  • （规范）ul里面只能放li，但li里可以放h* 和p等标签，dl里可以放dt和dd，dt和dd同级

  • 标签 : 将表单内容分组打包,生成一组相关表单的字段,特殊边框, 标签 : 为filedset元素定义标题

  • 不可继承的属性 :

    ① display ② margin ③ border ④ padding

    ⑤ background ⑥ height/width ⑦ position

Html结构

  • HTML: (Hyper Text Markup Language)超文本标记语言,用于构建网页基本结构及其内容
  • 超文本: 文本中指向其他文本的链接
  • 标记语言: 将文本及其他信息结合起来,展现出文档结构,如Html/Xml/Markdown
  • !DOCTYPE 标签: 他是指示web浏览器关于页面使用哪个HTML版本进行编写的指令

HTML5: 主要解决的问题

  1. 文档结构的混乱: 以前的文档结构过度依赖div标签,Html5则提出了多种语义化标签,使得文档易理解,并有利于seo优化
  2. 解决了浏览器之间的兼容性问题
  3. 提供了一些新的web应用的api

  • :刚html文档最前面位置,加上后就会按照W3C的HTML5标准啦解析渲染页面

  • <head> :对用户不可见,包括字符编码声明(utf-8 : 表示页面可用任何语言)、关键字、css样式、页面描述等等

块级元素

  1. 能够设置宽高，默认和父容器宽度一样
  2. 独立成行，(块级元素们)在页面从上往下排列，margin和padding均有效
  3. 可容纳其他块级元素、或内联元素
  4. 代表性的就是div，其他如p、h1-h6、nav、aside、header、footer、section、article、ol、ul-li、address、form、blckquote等等，都可以用div来实现。不过为了可以方便程序员解读代码，一般都会使用特定的语义化标签，使得代码可读性强，且便于查错。

行内(内联)元素

  1. 只占据它对应标签的边框所包含的空间,(不会自动换行) ( 非独立成行 )
  2. 只能容纳文本或其他内联元素
  3. 设置宽高无效
  4. 对margin仅设置左右方向有效，上下无效；padding设置上下左右都有效，
  5. 常见的行内元素:a、span、br、i、em、label、var、strong

行内块级元素

  1. 非独立成行

  2. 能设置宽高

  3. 在行内从左往右排列

  4. 常见行内块元素 ： img、input

  • 块级元素与内联元素区别： （ 块级元素）单独成行，可设置高度、宽度和外边距，

盒模型

（div、article、section）

① 标准盒模型 box-sizing : content-box

​ 此时盒子的宽和高只决定元素的content部分,即 width = content

② IE盒模型 box-sizing : border-box

​ 此时盒子的宽和高决定元素的border大小,即 width = content+padding*2+border*2

H5新特性

  1. 新增选择器 document.querySelector、document.querySelectorAll
  2. 媒体播放的 video 和 audio
  3. 本地存储 localStorage 和 sessionStorage
  4. 语意化标签 article、footer、header、nav、section
  5. 多任务 webworker
  6. 全双工通信协议 websocket
  7. 历史管理 history
  8. 跨域资源共享(CORS) Access-Control-Allow-Origin
  9. 绘画 canvas

html语义化

  • 脱掉CSS的外衣,依然头是头,脚是脚。还是一篇完整的文档。这就是语义化之后文档的效果

​ 语义化： 也无非就是在使用标签的时候多使用有英文语义的标签。

① header : 头部 ② slider : 侧边栏 ③ main : 主体 ④ footer : 底部

  • 语义化优点：
  1. 易于用户阅读，样式丢失的时候能让页面呈现清晰的结构。
  2. 有利于SEO，搜索引擎根据标签来确定上下文和各个关键字的权重。
  3. 方便其他设备解析，如盲人阅读器根据语义渲染网页
  4. 有利于开发和维护，语义化更具可读性，代码更好维护，与CSS3关系更和谐。

弹性布局

  • 弹性盒子是一种用于按行或者按列布局元素的一维布局方法

正常文档流

  • 默认的,按照书写顺序从上往下排,每个块级元素都会自动换行.如果有margin就会被隔开
    (p标签有默认margin)

脱离文档流

  1. 文档流: 将窗体自上而下分成一行一行，并在每行中按从左至右依次排放元素，称为文档流，也称为普通流。
    HTML中全部元素都是盒模型，盒模型占用一定的空间，依次排放在HTML中，形成了文档流。

  2. 元素脱离文档流之后，将不再在文档流中占据空间，而是处于浮动状态（可以理解为漂浮在文档流的上方）。
    脱离文档流的元素的定位基于正常的文档流，当一个元素脱离文档流后，依然在文档流中的其他元素将忽略该元素并填补其原先的空间

  • 方法:
  1. 使用float脱离文档流时，其他盒子会无视这个元素，但其他盒子内的文本认为这个元素依旧存在,会为这个元素让出位置
  2. 使用absolute脱离文档流后的元素，是相对于该元素的父元素进行定位的;
    并且这个父类元素的position必须是非static定位的
  3. fixed: 完全脱离文档流，相对于浏览器窗口进行定位。（相对于浏览器窗口就是相对于html）。

浮动布局

  • HTML元素分类:

  1. 块元素(可以设置宽度和高度,独立成行):       h1-6、p、div、ul、li

  2. 行内元素(内联元素): (不可以设置宽度和高度,不独立成行):   a、span

  3. 行内块元素(可以设置宽度和高度,独立成行)： img、imput、button

  4. 以上通过dispaly属性实现： block、inline、inline-block，还有一个none实现隐藏

清除浮动

原因: 浮动溢出:容器的高度不能自动伸长以适应内容的高度，使内容溢出到容器外而影响布局的现象。

方法:

  1. 使用带clear属性的空元素 :
    在浮动元素后使用一个空元素如 <div class="clear"></div>
    并在CSS中赋予.clear{clear:both;}属性即可清理浮动。
    亦可使用<br class="clear" />或<hr class="clear" />来进行清理。
  2. 使用CSS的overflow属性:
    给浮动元素的容器(浮动的它的父级)添加overflow:hidden;或overflow:auto;可以清除浮动
    因为overflow.hidden会触发BFC。 BFC的意思是，我这个元素里面的子孙元素，不会影响外部元素的布局。
    故该容器里需全为浮动的元素
  3. 给浮动的元素的容器添加浮动:
    给浮动元素的容器也添加上浮动属性即可清除内部浮动，但是这样会使其整体浮动，影响布局，(不推荐使用)
  4. 使用邻接元素处理:
    给浮动元素后面的元素添加clear属性。
  5. 使用CSS的:after伪元素:
    在元素末尾添加一个内容为一个点并带有 clear: both 属性的元素

iframe有什么优点、缺点

  • 优点：
  1. iframe能够原封不动的把嵌入的网页展现出来。
  2. 如果有多个网页引用iframe，那么你只需要修改iframe的内容，就可以实现调用的每一个页面内容的更改，方便快捷。
  3. 网页如果为了统一风格，头部和版本都是一样的，就可以写成一个页面，用iframe来嵌套，可以增加代码的可重用。
  4. 如果遇到加载缓慢的第三方内容如图标和广告，这些问题可以由iframe来解决。
  • 缺点：
  1. iframe会阻塞主页面的onload事件；
  2. iframe和主页面共享连接池，而浏览器对相同域的连接有限制，所以会影响页面的并行加载。会产生很多页面，不容易管理。
  3. iframe框架结构有时会让人感到迷惑，如果框架个数多的话，可能会出现上下、左右滚动条，会分散访问者的注意力，用户体验度差。
  4. 代码复杂，无法被一些搜索引擎索引到，这一点很关键，现在的搜索引擎爬虫还不能很好的处理iframe中的内容，所以使用iframe会不利于搜索引擎优化（SEO）。
  5. 很多的移动设备无法完全显示框架，设备兼容性差。
  6. iframe框架页面会增加服务器的http请求，对于大型网站是不可取的。
