Ajax

Asynchronous Javascript And XML（异步JavaScript和XML），
不过现在主流的数据格式是json

发送请求到后台,获取显示数据且不全局刷新的操作叫做AJAX

ajax是一种无需重新加载整个网页的情况下，能够部分更新页面内容的新方法,
ajax不是新的编程语言,而是多种技术的综合使用

  • 优点 : 按需获取数据,提升系统性能
  • 缺点 : 异步获取数据,不利于搜索引擎优化

全局刷新和局部刷新

  • 全局刷新: 使用form,href等发起的请求是全局刷新
    用户发起请求,视图改变了,旋转试图,使用新的数据添加到页面
    缺点:
    1. 传递数据量比较大,占用网络带宽
    2. 浏览器需要重新渲染整个页面
    3. 用户体验一般
  • 局部刷新: 在当前页面中发起请求,获取数据,更新当前页面的部分dom对象,对视图部分刷新
    特点:
    1. 数据量比较小,在网络中传播速度快
    2. 更新部分的页面内容,浏览器不需要全部渲染视图
    3. 在页面中,可以做多个局部刷新

异步请求对象

在局部刷新中,使用异步请求对象,在浏览器内部发起请求,获取数据
异步对象可以向服务器端独立发起请求,传递参数,接受服务器返回的结果数据

  • 异步对象(XMLHttpRequest)是浏览器内部的一种js对象,各大浏览器都能支持异步对象的使用。

异步对象XMLHttpRequest接收

js中的一种对象,使用js语法创建和使用这个对象。
var xhr = new XMLHttpRequest();
之后就可以使用xhr对象的属性或者函数,进行异步对象的操作
局部需要使用到的技术:

1、js：创建XMLHttpRequest对象,调用它的属性或方法
2、DOM：处理DOM，更新select的数据
3、CSS：处理视图，更新、美化。
4、sevlet：服务器端技术
5、数据格式：json （之前是xml）

以上这些的综合使用就成为Ajax:
使用js操作xhr异步对象,和服务器端交换使用json数据格式

异步

  • 同步: 同一时间只能做一件事(one by one),不能关闭界面,否则会中断,比如打电话

  • 异步: 系统中断,当前进程外部的实体可以触发代码执行,比如海王聊天(效率更高)

异步对象XMLHttpRequest属性和方法

  • 创建异步对象,使用js的语法
  • const xhr = new XMLHttpRequest();
  • XMLHttpRequest方法

open(请求方式,服务端的访问地址,异步还是同步(布尔值))

    • 例如 xhr.open(“get”,”http://127.0.0.1:3000/data",true) :
      表示我们向http://127.0.0.1:3000/data地址发送一个异步请求,用的是get方式

send(): 使用异步对象发送请求

    • xhr.send();

readyState属性:请求的状态

0 (未初始化)： (XMLHttpRequest)对象已经创建，但还没有调用open()方法。
1 (载入)：已经调用open() 方法，但尚未发送请求。
2 (载入完成)： 请求已经发送完成。
3 (交互)：可以接收到部分响应数据。
4 (完成)：已经接收到了全部数据，并且连接已经关闭。

status: 网络的状态,和Http的状态码对应

200: 请求成功
404:服务器资源没有找到
500: 服务器内部代码有错误

  • responsText属性: 表示 服务器端返回的数据
    例如: var data = xhr.responsText;

Promise的状态变化

  • promise实例对象中的一个属性 : PromiseState, 有三种可能的值

    • Pending状态（进行中）

    • Fulfilled/Resolved状态（已成功）

    • Rejected状态（已失败）

    • 一旦状态改变就不会再变 （两种状态改变：成功或失败）

      • Pending -> Fulfilled
      • Pending -> Rejected

  • promise实例对象的另一个属性 PromiseResult , 保存着异步任务 成功/失败的结果

    • 只有resolve和reject和throw可以改变实例中该属性的值,无论异步任务是否成功, 值(reason,value)都会被传到 PromiseResult 中

  • Promise构造函数接受一个执行器函数作为参数, Promise(executor){ } ,

    执行器 (resolve,reject)=>{ } ,执行器函数的两个参数分别是resolve和reject ,执行器在promise内部是立即同步回调，异步的操作是在执行器中另行执行的

  • Promise.prototype.then方法 : (onResolved, onRejected)=>{ }

    • onResolved 函数 : 成功时的回调函数 (value)=>{ }
    • onRejected 函数 : 失败时的回调函数 (reason)=>{ }
    • 返回一个promise, PromiseState : 执行的成功/失败, PromiseResult : 和成功/失败函数返回的promise的执行结果

  • Promise.resolve( )方法 : 返回一个成功/失败的Promise对象

    ```js
    let p1 = Promise.resolve(995);
    //传入参数为非Promise类型的对象,则返回结果为成功promise对象
    console.log(p1); //Promise {: 995}
    let p2 = Promise.resolve(new Promise((resolve, reject) => {

     //传入参数为Promise类型的对象时,返回结果就是传入promise对象的结果

    //resolve(‘OK’); //Promise {: ‘OK’}
    reject(‘Error’); //Promise {: ‘Error’}
    }))
    p2.catch((reason) => { //Promise失败,处理结果
    console.log(reason);
    })
    console.log(p2);

  • Promise.reject( )方法 : 永远返回一个失败的Promise对象

    传入什么参数,得到是 失败的且结果为那个参数的Promise对象

  • **Promise.all( )**方法 : 参数为包含n个promise的数组,返回结果为一个新的promise,只有参数中所有的promise都成功,结果promise才成功,此时PromiseResult中存放一个数组保存所有参数promise的结果。只要有一个失败,结果promise直接失败

  • Promise.race()方法 : 使用返回的第一个promise 的值作为值

  • async函数的用法 : async function foo() { }; //函数声明

    无论async函数有无await操作，其总是返回一个Promise。

    • 没有显式return，相当于return Promise.resolve(undefined);

    • return非Promise的数据data，相当于return Promise.resolve(data);

    • return Promise, 会得到Promise对象本身

  • await操作符用法 : let res = await expression;

    • expression是Promise，res等于Promise成功的值，若Promise被拒绝，则抛出异常，由try…catch捕获
    • expression是非Promise，会被转换为立即resolve的Promise，res等于expression
    • await操作只能用在async函数中，否则会报错。

Ajax原理

  • http概述
    http协议全称超文本传输协议,简单理解成为一个服务器与客户端通信的协议。

在http协议的约定下，客户端可以向服务器发送http请求，
服务器在接收到请求之后，给予客户端http响应(index.html)。

web服务器会给每种要通过web传输的对象都打上名为MIME类型的数据格式标签,
当Web浏览器从服务器取回一个对象时,会查看相关的MIME类型,来决定如何处理
MIMIE类型是一种文本标记,表示一种主要的对象类型和一个特定的子类型,用/来分隔

  • Ajax原理
    我们通过下面的方法向服务器发送请求
  1. 在浏览器直接输入URL
  2. a标签实现的页面跳转
  3. 表单提交
  4. Postman模拟http请求

在Ajax的请求原理: 通过XMLhttpRequest对象向服务器发送请求,

  • http协议请求的常用方法
  1. get方法
  2. post方法
  3. put方法 :将来自客户端的数据存储到一个命名的服务器资源中去
  4. delete方法

四种方法用来完成数据的增删查改 等操
真正的操作是服务器来完成

  • http协议状态码
    每条http响应报文返回时都会携带一个状态码
    状态码为客户端提供一种事物处理结果的便携方式,我们在network
    工具中可以看到响应头中的状态码
  1. 100-199(信息性状态码): HTTP/1.1向协议中引入了信息性状态码
  2. 200*299(成功状态码)
  3. 300*399(重定向状态码) : 到其他地方去获取资源
  4. 400-499(客户端状态码)
  5. 500-599(服务器状态码)
  • 3个常见HTTP状态码
  1. 200 OK: 请求成功
  2. 404 Not Found: 服务器资源没有找到
  3. 500 Internal Server Error: 服务器内部代码有错误
  • URI
    服务器资源名被统一称为统一资源标识符(Uniform Resource Identifier,URI)
    URI就像因特网上的邮件地址,在世界范围内唯一标识并定位信息资源

URI分为 URL (统一资源定位符)和 URN (统一资源名)

  • URL
    几乎所有URI都是URL,标准格式如下:
  1. 第一部分: 方案 (scheme): 说明了访问资源所使用的协议类型(通常是HTTP协议:http://)
  2. 第二部分: 给出了服务器的因特网地址(比如www.baidu.com)
  3. 其余部分制定了Web服务器上的某个资源(比如/images/yyds.jpg)
  • Postman
  • *Postman**可以模拟浏览器向服务器发送请求,
    在使用Postman之前，我们需要先完成服务器端程序，来处理数据的增删改查，

let fruit = ctx.request.body.fruit; //可以获取到页面中传入的新元素

let id = ctx.params.id; //可以获取页面中的id
传入id到后台:URL输入 /fruits/:id

跨域请求

  • 同源策略
    浏览器的一个安全功能,不同源的客户端脚本在没有明确授权的情况下,不能读写对方的资源比如432666.top采用ajax读取432.xyz里的文件数据会被拒绝

  • 相同的ip(域名),且同端口,则为同源否则为不同源
    同源策略限制了从同一个源加载的脚本如何与龙一个源的资源进行交互,
    这是一个隔离潜在恶意文件的重要安全机制

  • jsonp原理

ajax默认不能跨域,但script标签可以通过src属性获取到跨域的js文件
比如,我们想要在域名B获取域名A里的数据,可以在域名A的js文件里装载数据,然后再域名B通过script标签跨域引入当当前项目,进而使用数据
注意jsonp本质上并不是Ajax,只是功能很像

  • jQuery实现跨域请求数据
    前端:
    先引入jq

$(“.get”).click(() => {
$.ajax({
url: “http://127.0.0.1:3000/data",
dataType: “jsonp”,
}).done(res => {
console.log(res);
})
})
后台:

  1. const jsonp = require(“koa-jsonp”);
  2. app.use(jsonp());
  3. router.get(“/data”, ctx => {
    ctx.body = “Yoimiya”;

  • http-server
    在某个文件处cmp后使用http-server即可在8080打开这个文件

不通过jq,通过axios跨域

  • 后台:

  2. 在/data处的get请求内添加 ctx.set(“Access-Control-Allow-Origin”, “http://127.0.0.1:8080"); 后面的url可改为*,即任何地址都可跨域访问此data
  3. koa2-cors中间件,(cnpm,引入,use)
  4. 如果要引入parser并发送post请求,需要在路由之前进行parser的引入(否则报错!)

    引入axios
    axios.get(“http://127.0.0.1:3000/data").then(res => {

  })
