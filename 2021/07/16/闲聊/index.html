编程语言分类

  • 动态类型 : Pyhton、Ruby、PHP、JavaScript
    (运行时才会检查数据类型,不用给变量指定数据类型)
    (第一次赋值给变量时,才会记录其数据类型)(let a)
  • 静态类型:C、C++、C#、JAVA
    (编译期间检查数据类型,使用数据前,要声明数据类型)(int,float等)
  • 解释性语言 : 每次运行重新翻译,不是直接翻译成机器语言,而是先翻译成中间代码,再由解释器对中间代码进行解释运行,Pyhton、Ruby、PHP、JavaScript
  • 编译性语言 : 翻译发生在程序运行之前，将高级语言翻译成机器语言。再次运行时，可直接使用上一次翻译好的机器码，不需要重新编译。C / C++ / swift
  • 强定义类型定义语言
  1. 类型安全
  2. 强制数据类型定义的语言,如不强制转换,变量永远是某个指定的数据类型
  3. 比如 C/C++/JAVA/C#
  4. 举例: 整形变量不能赋值成字符串
  • 弱定义类型定义语言
  1. 类型不安全
  2. 数据类型可以忽略的语言,一个变量可以赋不同类型的值(boolean、str、number等)
  3. 比如 PHP/SQL/JavaScript/Python
  4. 举例: let a =1; a=”Yoimiya”;
  • 汇编语言无类型

关于log和dir和document.write

console.log(): 用于控制台调试，可以打印对象的内容,无法看到标签对象的属性和方法,
能输出多个数据，可以识别数据类型(颜色有差异)
console.dir(): 以对象形式输出标签对象的属性和方法,不能直观识别数据类型

document.write: 不用于调试，只能打印出对象类型,(在网页上写东西),Javascript 中只能在 HTML 输出流中使用 document.write(实际上文档加载时就是一边加载一边用 document.write 写出内容到屏幕上，而加载完成后，document 就关闭)，在文档已加载后(注意)使用它（比如在函数中），会覆盖整个文档(重写,html里只有空head和新body,故background也会被重置,)。

编译原理

var a = 2 如何执行:

编译器先编译到var a ,编译器会询问该作用域中是否有a,
如果有a,则不进行重复声明,编译下一步,否则会在该作用域中声明一个新变量,命名为a
编译到a=2,编译器会为引擎生成能够完成这个赋值操作的代码,引擎会询问作用域中是否有a,直至找到a并赋值给它,否则会报错

数组和类数组

  • 数组是一个特殊对象,它与普通对象的区别 :

    • 有新元素添加到列表中时，自动更新length属性
    • 设置length属性，可以截断数组
    • 从Array.protoype中继承了方法
    • 类型为’Array’(普通对象是Object)

  • 类数组是一个有length属性,且其他属性的值为非负正数的普通对象,不能直接调用数组方法

    与数组的区别 : 类数组是简单对象，它的原型关系与数组不同。

  • 类数组转换为数组

    • Array.from()
    • Array.prototype.slice.call()
    • Array.prototype.forEach() 进行属性遍历并组成新的数组
    • 须知:
      • 转换后的数组长度由 length 属性决定。索引不连续时转换结果也是连续的，会自动补位(为undefined)。
      • 仅考虑 0或正整数 的索引
      • 使用slice转换时,若有空位,为empty

深浅拷贝与赋值

赋值:(a = [1,2,3], b=a,) 赋值操作(包括对象作为参数、返回值),不会开辟新的内存空间,他只是赋值了对象的引用(也只对对象进行操作).也就是除了b这个名字之外,没有其他的内存开销,修改了a也就影响了b,修改了b,也就影响了a.

浅拷贝：只复制指向某个对象的指针而不复制对象本身，新旧对象还是共享同一块内存。
如果属性是基本类型，拷贝的就是基本类型的值(基本数据类型互不影响)；如果属性是内存地址（引用类型），拷贝的就是内存地址 ，因此如果其中一个对象改变了这个地址，就会影响到另一个对象。

深拷贝会另外创造一个一模一样的对象，新对象跟原对象不共享内存，修改新对象不会改到原对象。

Git

# 1. 创建一个空的远程仓库：登录github -> new repository

# 2. 编辑区添加到暂存区： git add .

# 3. 暂存区提交到(工作区)分支： git commit -m “备注”

# 4. 创建远程主机名： git remote add origin xxx.git(远程仓库地址)

# 5. 将分支推送到远程仓库： git push -u origin master

带上-u 参数其实就相当于记录了push到远端分支的默认值，这样当下次我们还想要继续push的这个远端分支的时候推送命令就可以简写成git push即可。

# 6. 克隆项目： git clone xxx.git

# 7. 拉取项目代码: git pull

# 8. Git会到远程仓库的 foo 分支上，然后获取所有本地不存在的提交，放到本地的 o/foo 上 : git fetch origin foo

# 9. 合并当前分支与bugFix分支到新节点,且都是该节点的父节点: git merge bugFix

# 10. 拉取项目代码: git pull

# 11. 新建分支: git branch foo1

# 12. 切换分支: git checkout foo1

# 13. 新建并切换分支: git checkout -b foo1

总结

  • 路由传递参数( 对象写法 ) path是否可以结合params参数一起使用?

    答: 单独写path不写name时, 不能使用params参数

  • 已在index.js的路由配置中进行接收params参数占位后,若不传params参数,跳转后,路径会出问题

    答: 在占位时,在参数后添加一个问号,代表可传可不传

  • 在params中传入空串时,也会有路径问题

    答: 使用undefined,即传参时传入 'xxx' || undefined

  • 路由组件如何传递props数据? 答: 可以

    • 布尔值写法 : 在index.js中,添加props:true配置,即可在路由组件中用props:['xx']获取外部的params属性,该方法只能传params参数

    • 对象写法: 配置props为对象,进行额外传参,如props:{a:1,b:2}

    • 函数写法 : 可以把params参数和query参数传递给路由组件
