命令行工具概述

  • 我们对计算机的操作几乎都是基于图形界面完成的,比如新建目录、运行某个程序，这些操作在计算机底层都是一个个命令，计算机接收到指令，做出相应反映。
  • 而通过命令行工具，我们不经过任何UI界面就可以给计算机下达命令。
  • cd..可以返回上一级
  • tab可以补全: cd D此时按下tab会补全为desktop,cd Do此时按下tab会补全为document
  • 直接D:回车就可以进入到D盘
  • 在图形界面按住shift点击右键可以打开powershell(win10)

什么是node

  • Node是一个基于ChromeV8引擎的JavaScript代码运行环境。
  • 通过node,让js具备后台开发的能力
  • 组成: Node.js是由ECMAScript+Node模块API组成。
  • 语法: 所有ECMAScript语法在Node环境中都可以使用。
  • npm是node的包的管理器,下载的第三方模块放在node_modules里,
  • npm服务器在国外,淘宝为国内用户提供cnpm,它是npm的镜像,可以代替npm

模块化开发

  • 实际开发中不可能把所有代码全写在一个文件夹中,所以node为我们提供了模块化的语法,每一个js文件都可以当做一个模块:
  1. require()  引入了一个外部模块
  2. module.exports 暴露模块接口
  • es6的模块化语法(与node模块化略有差异):
  1. import 引入外部模块
  2. export 暴露模块接口

外部模块的分类

  1. 核心模块: node自带的模块,可以在require引入后直接使用
  2. 自定义模块: 我们自己编写的模块,引入时需要完整的路径
  3. 第三方模块: 使用npm下载的就是第三方模块,下载完成后require直接引入后使用

常用的核心模块

  • fs模块:
    node 的文件系统模块,通过此模块的readFile方法可以读取文件

具体操作

  • 在需要导出的js中: exports.自定义名1=函数名; //注意此处别加括号
    其中函数名与上方定义函数的表达式使用的变量名相同,必须是表达式,不能是声明

  • 在需要引入的js中: const 自定义名2 = require(“./导出模块的js文件名”);
    let 结果变量 = 自定义名2.自定义名1(参数,参数);

web服务器

  • 1.客户端与服务器
    输入域名(比如http://baidu.com)后,按下回车,浏览器就会向百度服务器发送一个请求,服务器接到请求之后,会把百度的首页内容以文本的方式发送给浏览器(其中包括html、css、js)，然后我们看到就变成了网页。

几个基本概念：

  1. 请求：浏览器向服务器要数据
  2. 响应：服务器给浏览器发送数据
  3. 地址：我们可以通过域名或ip访问到一个网站，域名或者ip就是这个网站的地址
  4. 端口：一个ip或者一个域名可以找到一台服务器，但是这台服务器可以对外服务多个网站，他们的端口是不同的，因此访问一个站点除了输出ip或域名，还要输入端口，平时我们很少输入端口是因为几乎所有的网站都会使用默认的80端口，因此不必输入
  • 创建服务器

const http = require(‘http’);
const server = http.createServer((req, res) => {
res.end(‘Hello World’);
});
server.listen(3000, () => {
console.log(Server is running);
});

一般不这么实现node项目,而是使用框架实现
通过http://192.168.50.153:3000可以访问该页面
全局安装nodemon后,命令行输入nodemon server.js,可实现实时刷新

  • 框架实现

const Koa = require(“koa”);
const app = new Koa();
app.listen(3000,() => {
console.log(“server is running”)
});

引入的Koa是一个构造函数，通过new关键字可以创建一个koa应用app。然后通过app的listen方法设置监听端口，运行这个程序就可以移动一个koa服务器了。

cookie与session

  • 概述
    cookie:
  1. 商店给会员发的会员卡(不需要服务器接入)、
  2. 或者条形码(需要服务器接入进行店铺加密) :
    通过session对应cookie,完成验证
  • 设置cookie
    概述: 网站服务器程序可以在浏览器中写入cookie,然后浏览器再次访问这个网站时,就会带着这个cookie

ctx.cookies.set(“user”, “Yoimiya”); //此处属性值可为number型

服务器返回(reponse)cookie以后,客户端拿到会员卡,下次来(request)以及之后就一直保存有该cookie
哪怕访问此网站的其他页面,也一直携带该cookie

  • 获取cookie

router.get(“/test”, async ctx => {
let user = ctx.cookies.get(“user”);
ctx.body = user;
})

//此处为cookie字段名(属性名)

  • cookie过期时间
    //设置cookie过期时间为两秒(距离上一次重写cookie的时间超过两秒就重新载入cookie)

ctx.cookies.set(“count”, count, {
maxAge: 2000
})

  • session
    需要用到”koa-session”引擎
    //查阅所有npm下载的包可以去npmjs.com
    //nunjucks里的if语句用的是jinja的语法,自行查阅
    了解到不用get,都是直接 ctx.session.user就能获取到session
    session记录网页访问次数:

router.get(“/session”, async ctx => {
if (ctx.session.count > 0) {
ctx.session.count = ++ctx.session.count;
} else {
ctx.session.count = 1;
}
ctx.body = ctx.session.count;
})

  • 登录验证(通过if语句)

router.post(‘/login’, async ctx => {
let username = ctx.request.body.username;
let password = ctx.request.body.password;

if (username !== '' && password === "yyds") {
    ctx.session.user = "yyds";
    ctx.redirect("/content");
} else {
    ctx.redirect("/");
}

})

  • 登录才能查看内容页

router.get(“/content”, async ctx => {
if (ctx.session.user) {
await ctx.render(“content”);
} else {
ctx.redirect(“/login”);
}
})

    页面内逻辑实现网页内部跳转,即为重定向
    重定向不需要await 直接ctx.redirect()
