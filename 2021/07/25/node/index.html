命令行工具概述

  • 我们对计算机的操作几乎都是基于图形界面完成的,比如新建目录、运行某个程序，这些操作在计算机底层都是一个个命令，计算机接收到指令，做出相应反映。
  • 而通过命令行工具，我们不经过任何UI界面就可以给计算机下达命令。
  • cd..可以返回上一级
  • tab可以补全: cd D此时按下tab会补全为desktop,cd Do此时按下tab会补全为document
  • 直接D:回车就可以进入到D盘
  • 在图形界面按住shift点击右键可以打开powershell(win10)

什么是node

  • Node是一个基于ChromeV8引擎的JavaScript代码运行环境。
  • 通过node,让js具备后台开发的能力
  • 组成: Node.js是由ECMAScript+Node模块API组成。
  • 语法: 所有ECMAScript语法在Node环境中都可以使用。
  • npm是node的包的管理器,下载的第三方模块放在node_modules里,
  • npm服务器在国外,淘宝为国内用户提供cnpm,它是npm的镜像,可以代替npm

模块化开发

  • 实际开发中不可能把所有代码全写在一个文件夹中,所以node为我们提供了模块化的语法,每一个js文件都可以当做一个模块:
  1. require()  引入了一个外部模块
  2. module.exports 暴露模块接口
  • es6的模块化语法(与node模块化略有差异):
  1. import 引入外部模块
  2. export 暴露模块接口

外部模块的分类

  1. 核心模块: node自带的模块,可以在require引入后直接使用
  2. 自定义模块: 我们自己编写的模块,引入时需要完整的路径
  3. 第三方模块: 使用npm下载的就是第三方模块,下载完成后require直接引入后使用

常用的核心模块

  • fs模块:
    node 的文件系统模块,通过此模块的readFile方法可以读取文件

具体操作

  • 在需要导出的js中: exports.自定义名1=函数名; //注意此处别加括号
    其中函数名与上方定义函数的表达式使用的变量名相同,必须是表达式,不能是声明

  • 在需要引入的js中: const 自定义名2 = require(“./导出模块的js文件名”);
    let 结果变量 = 自定义名2.自定义名1(参数,参数);

web服务器

  • 1.客户端与服务器
    输入域名(比如http://baidu.com)后,按下回车,浏览器就会向百度服务器发送一个请求,服务器接到请求之后,会把百度的首页内容以文本的方式发送给浏览器(其中包括html、css、js)，然后我们看到就变成了网页。

几个基本概念：

  1. 请求：浏览器向服务器要数据
  2. 响应：服务器给浏览器发送数据
  3. 地址：我们可以通过域名或ip访问到一个网站，域名或者ip就是这个网站的地址
  4. 端口：一个ip或者一个域名可以找到一台服务器，但是这台服务器可以对外服务多个网站，他们的端口是不同的，因此访问一个站点除了输出ip或域名，还要输入端口，平时我们很少输入端口是因为几乎所有的网站都会使用默认的80端口，因此不必输入
  • 创建服务器

const http = require(‘http’);
const server = http.createServer((req, res) => {
res.end(‘Hello World’);
});
server.listen(3000, () => {
console.log(Server is running);
});
//一般不这么实现node项目,而是使用框架实现
//通过http://192.168.50.153:3000可以访问该页面
//全局安装nodemon后,命令行输入nodemon server.js,可实现实时刷新

  • 框架实现

const Koa = require(“koa”);
const app = new Koa();
app.listen(3000,() => {
console.log(“server is running”)
});

引入的Koa是一个构造函数，通过new关键字可以创建一个koa应用app。然后通过app的listen方法设置监听端口，运行这个程序就可以移动一个koa服务器了。

设置响应的内容

  • use()方法
    在创建应用之后,监听事件之前,使用use方法.
    //ctx：上下文变量，可以获取请求与响应的相关信息。

app.use(async ctx=>{
ctx.body=”welcome to my node world”;
})

路由知识

浏览器可以使用不同的方法向服务器发送请求,常用的如下:

  • get请求
    先cnpm一个koa-router包，随后引入并执行

const router=require(“koa-router”)();

//注意加一个括号表示执行
//然后就可以使用get请求了

router.get(‘/‘,async ctx=>{
ctx.body=<h1>Yoimiya</h1>;
})

使用get就不需要每次单独使用use了,只需要在get最后

app.use(router.routes());

模板引擎

作用
可以直接*设置响应的html页面,并且可以把后台数据绑定到模板中,发送给客户端
我们选择一个易上手的模板引擎:nunjucks
创建应用之后,监听之前:

*操作(模板引擎的单独步骤)

const nunjucks = require(“nunjucks”); //引入模板引擎
const views = require(“koa-views”); //指定模板引擎
app.use(views(__dirname + “/views”, { map: { html: “nunjucks” } }
)) //此处两个参数,第一个指定模板路径,第二个指定模板引擎,都需要在views里
router.get(“/“, async ctx => {
await ctx.render(“server”, { title: “welcome to my node” })
//结合路由,引入title变量到模板,在html里使用来使用
})

  • input的name属性:定义表单提交的数据字段

*form的属性method: 用什么方法请求(默认get)

method的值: GET(获取)、 post(发送)、

  • get方法:(server.js里要有router.get) 传表单数据到页面里

router.get(“/login”, async ctx => {
let username = ctx.query.username;
let password = ctx.query.password;
await ctx.render(“home”, { username: username, password: password })
})
