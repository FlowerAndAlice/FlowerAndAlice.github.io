MVC概述

  • 模型(model)-视图(view)-控制器(controller)的缩写

  • MVC模型是一种软件设计规则,主要作用就是逻辑拆分:

  1. 视图为用户展示数据
  2. 控制器用来处理用户输入
  3. 模型用于处理数据

Egg中的controller(控制器)

  1. 直接响应数据或渲染模板
  2. 接收用户的输入
  3. 与路由建立对应关系

路由发送get请求

  • 获取query参数:

let query = this.ctx.request.query;
再通过值:${query.index}获取query对象的属性

  • 获取params参数:

//在router.js里:
router.get(‘/prays/:id‘, controller.prays.getId)
(加一个:id, 再把方法改为新方法)

然后再在pray.js(自己的路由)里

随后在路由里加一个/id即可获取到id参数

路由发送post请求

  • egg里对post请求做了一些安全验证,在config.ddefault.js中,这样设置即可
    config.security = {

      csrf: {
      enable: false,
  },

    };

  • 先写一个方法创建表单,method=’post’ action=’/该路由’ 里面的输入input有name=”某属性”

  • router.get调用该路由及该方法

  • router.post调用该路由和一个新方法

  • 在新方法里通过this.ctx.request.body获取post得到的参数,(并将它push到本地数据里)(不需要this)(获取的是对象,故需push(参数.某属性名))

  • 回到本地(this.ctx.body=prayList的)路由

  • 以上这种方法太麻烦,可通过restful风格的url简化路由文件
    (在router.js中) router(‘接口名’,’接口url’,controller.接口名)

  • (接口中)写出方法后,通过访问对应路由即可调用该功能

用户登录状态

  • 需要3个路由:

    1. login表单(action=”/login”)的get路由,对应login方法
    2. login表单的post路由,对应doLogin方法
    3. logOut表单(模板中的包含注销的表单)(action=”/logOut”)的post路由,对应logOut方法

  • session

    1. doLogin方法中,通过this.ctx.request.body.username获取用户名

    2. 通过this.ctx.session.user = username; 可设置session

    3. 通过this.ctx.session.user = ‘’;可清空sessionJWT( Json Web Token )

验证流程

  1. 根据用户信息,生成token,并响应给客户端(前端)
  2. 客户端存储在localstorage中
  3. 客户端每次查看数据,请求头携带token
  4. 服务器接收请求时,验证请求头的token,验证成功则响应数据

解析token

我们使用了try……catch语句，如果token验证成功则执行try语句，如果token验证失败则会有个反馈
