vue引入

  1. script的src属性引入Vue.js文件
  2. 使用node环境,构建基于Vue的web项目 (一般项目开发采用这种)

引入Vue.js

  • 把js里的数据添加到DOM里,之前是先获取DOM节点(document.querySelector)再操作节点

  • 一个vue程序最基本的结构：

首先，我们在html中定义了一个id为app的容器，然后我们在js创建一个Vue的实例，通过el属性指定这个容器，从而实现让js接管html.

  • el属性:将vue挂载到html

vue中的前端依赖

  1. axios ： cnpm install –save axios

绑定文本

  • data属性: 在Vue的实例之下添加属性

  • {{}}: 可以输入一个表达式,也可以直接获取Vue实例的值

  • 能将vue实例里data中的message属性通过双花括号显示在文本节点之中，

绑定属性

  • 还能将数据绑定成为一个html属性(vue实例里data中的message属性)，使用【v-bind:属性名】和【:属性名】两种方式都可以绑定属性，在实际开发中，我们通常使用简写。

绑定事件

  • 在vue实例中可以设置methods属性为vue的示例添加方法，这些方法我可以直接被调用，也可以绑定为事件

  • 使用【v-on:事件类型】和【@事件类型】两种方式都可以为元素绑定事件，在实际开发中，通常使用简写。

  • 在methods中通过this.属性 可获取data中设置的属性

组件化开发

  • 以vue为后缀的文件时vue 的单文件组件
  • 组件可以理解为一个 可以自定义的, 有更强大功能的标签
  • 开发web项目时,其实就是在编写和组装这些组件

指令

  • 指令(directives)是带有v-前缀的特殊属性,v-bind:(简写:)(绑定属性)和v-on:(简写@)(绑定事件)都是指令

  • v-if和v-show都能控制元素的显示与隐藏

  • v-if=’false’：不会渲染DOM，查看元素不可见。

  • v-show=’false’：会渲染DOM，查看元素可见，但是样式为display:none;

  • v-for显示列表: 哪个要循环就把v-for写在哪个标签里

组件嵌套

  • 自定义组件一般在components目录中创建，采用大写字母开头+驼峰命名法。

  • 注意注册组件时不要把components打错! 使用下面代码可引入当前组件

组件传值

  • 父级向子级传值: 在app.vue 的data内写一个属性1 meessage,在子级的调用标签,添加 :属性2=”属性1”,在子级组件export内 通过 props: [“属性2”], 之后即可通过{{属性2}}使用该值

  • 插槽: (可自定义button等 )在父组件调用子组件的标签里通过template标签的v-slot属性可在定义对应子组件的slot对应name的插槽里自定义html内容

  • 子级向父级传值: 在app.vue内的子级的调用标签,添加自定义事件
    比如 @myEvent=”changeData” ,然后再html内定义要传的值得表达式,
    比如 <h3>{{childData}}</h3> ,随后在data内初始化要传的值,

  • 来到子级,先给传参绑定一个click事件,在该事件内,this.$emit(“myEvent”, this.childData)
    即可将子级的数据传到父级内的自定义事件的参数

  • 最后在父级的自定义事件把参数传到data里

  • 若是在子级组件里的template标签写div,则引入组件会因为块元素而换行

  • 只有 v.属性 才能获取对象里的属性

非父子级组件传值

  • 在src目录中创建一个store.js文件用来统一存储数据
  • 随后在要传入的组件先(import)引入store.js,再在data里传值:
    state : store.state, (注意此处是冒号)(即可将store里的数据传到组件)
  • 调用store里的setStateMessage方法可将设置的参数传给state.message

DOM操作

  • Vue中的数据变化,并不是直接改变DOM,而是通过改变虚拟DOM,并计算更改差异,进而修改DOM中有变化的内容
  • 通过doc.query..获取真实DOM之后,通过window.getComputedStyle(节点) 可获取节点样式并进行操作
  • 但vue提供了新的获取DOM节点的方法 (注意这里是虚拟DOM)
    (给节点绑定一个属性叫ref,然后通过let box=this.$refs.ref的值,即可获取到节点)

项目部署

  • 前端8080的变量要用 VUE_APP_变量名 来命名,可实现生产环境和开发环境的(ip/变量名)区分
  • 在前端新建两个文件: .env.development和.env.production (重启后生效)
  • 关闭8080后在命令行npm run build 可在根目录获得一个dist文件夹
  • 将dist文件夹里的html/css拷贝到server(后台)的public文件夹,并在config中设置目录为public
  • 在app.Vue里要用${process.env.VUE_APP_变量名}来使用该变量
  • 再运行后台可实现项目部署
