Vue的特点

  1. 采用组件化模式,提高代码复用率、且让代码更好维护
  2. 声明式编码（v-for），让编码人员无需直接操作DOM，提高开发效率 （相对的有麻烦的命令式编码）
  3. 使用虚拟DOM + 优秀的diff算法
  4. MVVM框架中Vue作为ViewModel进行Data bindings(数据绑定)和DOM Listener(DOM监听)( 发生事件时鉴定到且有时会改变相应Data )

实例生命周期钩子

  • 每个Vue实例在被创建时都要经过一系列的初始化过程，例如设置数据监听、编译模板、将实例挂载到DOM并在数据变化时更新DOM等。
  • 同时在这个过程中也会运行一些叫做生命周期钩子的函数,这给了用户在不同阶段添加自己的代码的机会
  • 常用的生命周期钩子:
    • mounted : 发送ajax请求、启动定时器、绑定自定义事件、订阅消息等初始化操作
    • beforeDestroy : 清除定时器、解绑自定义事件、取消订阅消息等收尾工作
  • 关于销毁Vue实例
    • 销毁后借助Vue开发者工具看不到任何信息
    • 销毁后自定义事件会失效，但原生DOM事件依然有效。（比如已经绑定成原生事件）
    • 一般不会在beforeDestroy操作数据，因为即时操作数据，也不会再触发更新流程了
  • 生命周期钩子中的this指向是vm或者组件实例对象
  • 不要将生命周期钩子写成箭头函数,可能导致作用域错误
  • 生命周期相关方法 :
    • vm.$nextTick(回调函数) ,下一次DOM更新后,执行指定回调,为了减少DOM更新,Vue把所有的更新放到下一次微任务执行,而该方法能将回调添加到微任务,故此执行回调时,DOM已更新

指令语法

  • 功能：用于解析标签（包括：标签属性、标签体内容、绑定事件…..）。
  • 指令(directives)是带有v-前缀的特殊属性,v-bind:(简写:)(绑定属性)和v-on:(简写@)(绑定事件)都是指令
  • 枚举: v-for,v-if,v-else,v-show
  • v-text : 向其所在的节点中渲染文本内容,且会替换原内容
  • v-html : 向指定节点中渲染包含html结构的内容, 特点:可以识别html结构,严重注意:v-html有安全性问题!,
    • 在网站上动态渲染任意HTML非常危险,容易导致XSS攻击
    • 一定要在可信的内容上使用v-html,永远不要在用户提交的内容上!( 还可以加上验证 )
  • v-cloak : 没有值,
    • 本质是一个特殊属性,Vue实例创建完毕并接管容器后,会删掉v-cloak属性
    • 使用CSS配合v-cloak ( 属性选择器[v-cloak]{display:none}) 可以解决网速慢时,js还在下载,页面展示出的问题
  • v-once : 没有值
    • v-once 所在节点在初次动态渲染后,就视为静态内容了
    • 以后数据的变化不会引起 v-once 所在结构的更新,可以用于优化更新
  • v-pre : 也没有值
    • 跳过其所在节点的编译过程 ( Vue不解析啦 )
    • 可以利用它跳过没有语法指令、没有使用插值语法的节点,会加快编译, ( 跳过了不需要vue编译的普通节点 )
  • v-model : 有值,值代表checked

自定义指令

  • 定义指令 :
    • 局部指令
      • new Vue({ directives:{指令名:配置对象/简写函数} })
    • 全局指令
      • Vue.directive("指令名",配置对象/简写函数 )
  • 一个指令对象可以提供如下几个钩子函数( 均为可选 ) :
    • bind : 只调用一次,指令第一次绑定到元素时调用
    • inserted : 被绑定元素插入父节点时调用( 插入页面 )
    • update : 所在组件的VNode更新时调用
      • ps : 指令定义时不加v-,但使用时要加v-
      • 指令名如果是多个单词,要是用kebab-case命名方式,而不能用驼峰,定义时要加引号,如 "big-number":花括号/fn

条件渲染

  • v-if 和 v-show 都能控制元素的显示与隐藏

  • v-if='false’：不会渲染DOM，查看元素不可见。(适用单次变化)

  • v-show='false'：会渲染DOM，查看元素可见，但是样式为display:none; ( 适用多次变化 ) ( 高频变化 )

  • template只能配合v-if使用,渲染时会脱去template的外衣

渲染列表

  • v-for显示列表: 哪个要循环就把v-for写在哪个标签里

  • 可遍历: 数组 (用的多),对象,字符串,指定次数(从1开始)

  • 使用for循环就必须使用:key绑定每项的唯一标识

  • 没拿到需要的数据时,v-for不显示, 拿到数据后,才会渲染并更新视图

key的作用与原理

  • 虚拟DOM中key的作用:

    • key是虚拟DOM对象的标识, 当状态中的数据发生变化时, Vue会根据 新数据 生成 新的虚拟DOM , 随后Vue进行新虚拟DOM 与 旧虚拟DOM的差异比较

    • 比较规则:

      • 旧虚拟DOM中找到了与新虚拟DOM相同的key:
        1. 虚拟DOM中内容没变的节点,直接使用之前的真实DOM!
        2. 虚拟DOM中内容改变了的节点,则生成新的真实DOM,随后替换掉页面中之前的真实DOM
      • 旧虚拟DOM中未找到与新虚拟DOM相同的key:
        • 创建新的真实DOM,随后渲染到页面

    • 用index作为key可能会引发的问题：

      • 若对数据进行：逆序添加、逆序删除等破坏顺序操作:会产生没有必要的真实DOM更新 ==> 界面效果没问题, 但效率低。

      • 如果结构中还包含输入类的DOM：

        会产生错误DOM更新 ==> 界面有问题。

    • 开发中如何选择key?:

      • 最好使用每条数据的唯一标识作为key, 比如id、手机号、身份证号、学号等唯一值。

      • 如果不存在对数据的逆序添加、逆序删除等破坏顺序操作，仅用于渲染列表用于展示，使用index作为key是没有问题的。

插值语法

  • data属性: 在Vue的实例之下添加属性

  • 两个花括号: 可以输入一个表达式,也可以直接获取Vue实例(vm)里的任意属性的的值,能将vue实例里data中的message属性通过双花括号显示在文本节点之中，因为data中的数据最终会出现在vm上,而vm和原型身上的所有属性都可以在花括号里使用

数据绑定

  • 单向绑定(v-bind),数据只能从data(vm)流向页面(view),
    • v-bind:href="xxx" 或 简写为 :href="xxx"，xxx同样要写js表达式， 且可以直接读取到data中的所有属性。
  • 双向绑定(v-model:value="xxx") , 数据在data(vm)和页面(view)双向流动,但v-model指令只能用在表单类元素(输入类元素)(得有value值)( input,单选框,复选框,多行输入.. )上
    • 可以简写为v-model:"xxx",因为v-model默认收集value值
  • 还能将数据绑定成为一个html属性(vue实例里data中的message属性)，使用【v-bind:属性名】和【:属性名】两种方式都可以绑定属性，在实际开发中，我们通常使用简写。
  • data中的数据变化会让vue重新解析模板
收集表单数据
  • 写法 v-model=”数据”
  • <input type="text"/>, 则v-model收集的是value值( 用户输入 )
  • <input type="radio"/>, value值要配置给标签,v-model的值与value值相等时,该单选框就被选中,选中后v-model的值就等于对应标签中value的值
  • <input type="checkbox"/>, 复选框 :
    • 没有配置value属性,则收集到的是checked,( 全选/全不选 )
    • 配置了value属性 :
      • v-model初始值是非数组,则收集的是checked(全(不)选)
      • v-model的初始值是数组,则收集的是value组成的数组,选中后,会在该数组尾部添加一项对应的value值( 有几个value值就勾选对应的框 )
  • v-model的三个修饰符
    • lazy : 失去焦点时再收集数据( 聚焦输入时不再收集 )
    • number: 输入字符串转为number类型,不再是String类型
      • 另input的type="number"后,input中只能输入数字
    • trim : 输入首尾空格过滤

绑定事件

  • 使用v-on:xxx 或 @xxx 绑定事件，其中xxx是事件名；
  • 事件的回调需要配置在methods对象中，最终会在vm上；
  • methods中配置的函数，不要用箭头函数！否则this就不是vm了；
  • methods中配置的函数，都是被Vue所管理的函数，this的指向是vm 或 组件实例对象；
  • @click="demo" 和 @click="demo($event)" 效果一致，但后者可以传参；

数据代理

  • vue中的数据代理: 通过vm对象来代理data对象中属性的操作(读/写)

  • 好处: 更方便的操作data中的数据(不再需要用_data.name调用)

  • 在vscode中编写new Vue后, 会新建个vue实例,并将data中的数据copy到vm的_data中

  • 原理: 通过Object.defineProperty()把_data中所有属性添加到vm上。为所有添加到vm上的属性, 都指定一个getter /setter。在getter / setter内部需操作( 读/写 ) data中对应的属性

事件修饰符

  1. prevent：阻止默认事件（常用）；
  2. stop：阻止事件冒泡（常用）；
  3. once：事件只触发一次（常用）；
  4. capture：使用事件的捕获模式；
  5. self：只有event.target是当前操作的元素时才触发事件；
  6. passive：事件的默认行为立即执行，无需等待事件回调执行完毕；
  7. 修饰符可以连写,比如@click.stop.prevent,既阻止冒泡又阻止默认事件

计算属性

  • 定义: 需要的属性不在data中,要通过已有的属性计算得来
  • 原理: 底层借助了Object.defineProperty方法提供的getter和setter
  • computed里的函数(getter)什么时候调用?
    1. 初次读取该计算属性时
    2. 所依赖的数据发生变化时
  • 优势: 与mothods相比,内部有缓存机制( 复用 ),效率高,方便调试
  • 不应该在模板中放入太多的逻辑,故对于复杂逻辑,需要使用计算属性,得到的是computed(一个对象)中totalPrice函数的返回值
    (多个值变化为了得到一个值,此时用计算属性) (为了得到一个值)
  • computed中的属性会被执行后,最终返回结果会出现在vm上,故通过直接读取即可
  • 备注:
    1. computed里的函数的this指向vue实例,即和模板一样,直接用数据
    2. 如果要修改计算属性,必去通过set函数去响应修改,且set中要引起计算时依赖的数据发生改动
    3. 不需要setter时简写为totalPrice(){ … }

计算属性computed和侦听器watch对比

  • 多个值的改变,为了得到一个结果,使用计算属性computed (别人变化影响自己) (由data中的已知值得到一个新的值)

    • 比如路飞的全名 : 蒙奇·D·路飞

    • 要计算的属性会随着其依赖的data中属性变化而变化，只要依赖的属性改变，就会执行computed，若无改变，则直接从缓存中取值

    • 支持缓存，只有依赖数据发生改变，才会重新进行计算

    • 不支持异步，当computed内有异步操作时无效，无法监听数据的变化

    • computed 属性值会默认走缓存

    • 只需要getter时,才可以简写为函数形式

  • 一个值得变化会影响多个值( 或处理多件事 ),使用侦听器watch

    • 比如 草帽海贼团(变化) +( 影响 )索隆/娜美/乔巴

    • 不支持缓存，数据变，直接会触发相应的操作；

    • watch支持异步；

    • 监听的函数接收两个参数，第一个参数是最新的值；第二个参数是输入之前的值；

    • 监听的是这个属性自身的变化，且不会操作缓存

    • computed能完成的功能,watch都能完成
    • watch能完成的,computed不一定能完成,例如异步操作
    • computed和watch都没有this
    • 注意: 被Vue管理的函数,最好写成普通函数,这样this才指向vm或者组件实例对象
    • 不被Vue管理的函数( 定时器的回调,ajax的回调函数等 ) ,最好写成箭头函数,这样this的指向又回到vm/组件实例了

生命周期

  • 每个vue实例在被创建时(new Vue)都要经过一系列的初始化过程,
    也会默认调用一些(vue内置定义的)函数,这些函数就是vue的生命周期函数(钩子)

  • created在页面加载之前执行,mounted则在模板之后
    (这两者只有调用异步函数之后,在异步函数里修改变量,且之后变量会复原)

  • 三个隐藏钩子: newxtTick(DOM更新) ,activated(路由组件激活),deactivated(路由组件失活)

  • 视图阶段

    • beforeCreate实例未生成，访问不到this，一般用于设置title，keywords等；也可以用于路由权限控制拦截；

    • create;组件实例生成，处于未挂载阶段；组件首次访问data，methods等；一般用于数据初始化，ajax请求；

    • beforemount template模板的编译，从而形成相应的render函数在；组件挂载前；这时实例依然未挂载，访问不到dom中的data

    • mounted template模板已经真实的挂载到了页面中，用户已经可以看到渲染好的页面了

  • 数据更新阶段

    • beforeUpdate 页面中的显示的数据，还是旧的，此时 data 数据是最新的，页面尚未和最新的数据保持同步

    • updated由于数据更改导致的虚拟 DOM 重新渲染和打补丁，在这之后会调用该钩子。当这个钩子被调用时，组件 DOM 已经更新，所以你现在可以执行依赖于 DOM 的操作。

  • 销毁阶段

    • beforeDestroy实例销毁之前调用。在这一步，实例仍然完全可

    • destroyed:组件的数据绑定、监听…都去掉了,只剩下dom空壳

    特殊钩子： activated 与 deactivated— keep-alive包含的组件

VueX

  • 概念: 专门在Vue中实现集中式状态(数据)管理的一个Vue插件,对vue应用中多个组件间的共享状态进行集中式的管理(读/写),也是一种组件间通信方式

  • Vuex 是一个专为 Vue.js 应用程序开发的状态管理模式 + 库。它采用集中式存储管理应用的所有组件的状态，并以相应的规则保证状态以一种可预测的方式发生变化。

  • 把组件的共享状态抽取出来，以一个全局单例模式管理, 在这种模式下，我们的组件树构成了一个巨大的“视图”，不管在树的哪个位置，任何组件都能获取状态或者触发行为！

  • 什么时候用VueX:

    • 多个组件依赖于同一状态,( 项目很大,组件很多,数据很多,维护 )
    • 来自不同视图的行为需要变更同一状态。

  • store组成: Action(发AJAX) + Mutations(处理state的数据) + State(存放状态)

    • store提供Dispatch方法给action

  • 搭建vuex环境

    • 组件中读取vuex中的数据: $store.state.sum

    • 组件中网络请求/业务逻辑 : $store.dispatch('actions中的方法名',数据)

    • 修改state中的数据: $store.commit('mutations中的方法名',数据),如果没有业务逻辑也可以跳过actions直接用commit
