简介

  • TS是JS的超集
  • 它对JS进行了扩展,向JS中引入了类型的概念,并添加了许多新的特性
  • TS代码需要通过编译器编译为JS，然后再交由JS解析器执行
  • TS完全兼容JS，换言之，任何的JS代码都可以直接当成TS使用。
  • 相较于JS而言，TS拥有了静态类型，更加严格的语法，更强大的功能
  • TS可以在代码执行前就完成代码的检查，减小了运行时异常的出现的几率；TS代码可以编译为任意版本的JS代码，可有效解决不同JS运行环境的兼容问题；同样的功能，TS的代码量要大于JS，但由于TS的代码结构更加清晰，变量类型更加明确，在后期代码的维护中TS却远远胜于JS。

预备知识

  • never是空集,字面量(“a”,1,ture)和null,undefined是单元素集合

  • Boolean={true,false}是有限集合

  • String,Symblo,BigInt和number={…}是无限集合(近似)

  • 还有一个全集

  • A|B 是 A或B (并集)(联合类型Union Types), A&B (交集)(交叉类型Intersection Types)

  • JS中用let等声明变量,TS中可用tpe为类型声明别名(采用块级作用域)

  • 用let和var声明变量时,比如let a=”Yoimiya”,TS认为变量会改变,(类型拓宽)故将类型推断为较为宽泛的类型string,但如果是const a=”Yoimiya”,TS知道常量不会改变,(类型收窄)故推断为字面量类型”Yoimiya”

  • 类型空间: 编译器存在的(比如type,interface的)各种类型,这个空间由TS的编译器tsc创建

  • 值空间: 运行时存在的(比如let,const的)各种值,有JS引擎(比如V8)

  • class等枚举会在两者都添加,instanceof只作用于值空间

  • typeof在值空间返回js类型,在类型空间返回TS类型(看等号左边确定所在空间)

  • bigint是新引入的类型(es2020+才能使用),表示任意大小的整数,(number范围:[-(2^53)-1,(2^53)-1])

  • bigint字面量是在数字后面加小写’n’,bigint不能和number混合运算,需要显示转换

  • 强类型的编程语言都有类型系统,分为:

  1. 结构类型系统:通过类型的实际结构确定两个类型是否相等或兼容(TS、Go、OCaml等)
  2. 名义类型系统：通过类型的名称确定两个类型是否相等(C、C++、JAVA等)

配置

  • cmd面板使用npm全局安装typescript: npm i -g typescript
  • 创建ts文件夹,src子文件夹(并在里面写一个index.ts)
  • (初始化ts项目)npm init -y
  • npm install –save-dev typescript
  • 在终端使用tsc init 命令生成tsconfig.json配置文件(或者在vscode打开ts文件,点击右下角ts版本号,也可创建tsconfig)
  • 在tsconfig.json设置include ,注意加””
  • 在include路径里创建ts文件
